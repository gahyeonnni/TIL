# 의존성 주입(Dependency Injection, DI)

### 의존성 주입(Dependency Injection) 이란?

- Spring 프레임워크는 3가지 핵심 프로그래밍 모델을 지원하고 있는데, 그 중 하나가 의존성 주입
    - 객체를 주입하기 위해서는 애플리케이션 실행 시점에 필요한 객체(빈)를 생성
    - 의존성이 있는 두 객체를 연결하기 위해 한 객체를 다른 객체로 주입
        
        ```sql
        Market market = new Market(apple);
        ```
        
    - 스프링은 특정 위치부터 클래스를 탐색하고, 객체를 만들며 객체들의 관계까지 설정
        - → 이러한 이유로 스프링은 DI 컨테이너라고도 불림!
        - 이러한 개념은 제어의 역전
            - 어떠한 객체를 사용할지에 대한 책임은 프레임워크에게 넘어갔고, 자신은 수동적으로 주입받는 객체를 사용
- DI란 외부에서 두 객체 간의 관계를 결정해주는 디자인 패턴
    - 외부에서 객체를 생성한 뒤 이를 전달해 의존성을 해결하는 방법
        - 객체 내부에서 직접 생성하는 경우가 아니라 생성 책임을 → 외부로 위임하고 외부에서 전달해주는 방식을 통해서는 결합도를 낮추는 유연한 설계 가능
    - 인터페이스를 사이에 둬서 클래스 레벨에서는 의존관계가 고정되지 않도록 함
    - 런타임 시에 관계를 동적으로 주입하여 유연성을 확보하고 결합도를 낮출 수 있게 해준다
        - 의존성
            - 한 객체가 다른 객체를 사용할 때 의존성
            - Class A가 Class B의 코드를 참조할 때, A는 B에 의존한다고 말한다

### 의존성 주입이 필요한 이유

**1. 두 클래스가 강하게 결합되어 있음** 

- 유연성이 떨어진다
- 상속을 사용하여도 상속은 제약이 많고 확장성이 떨어지므로 피하는 것이 좋다
1. 객체들 간의 관계가 아니라 클래스 간의 관계가 맺어짐

### 의존성 주입의 장점

- 두 객체 간의 관계라는 관심사의 분리
- 두 객체 간의 결합도를 낮춤
- 객체의 유연성을 높임
- 테스트 작성을 용이하게 함

### **의존성 주입의 단점**

- 책임이 분리되어 있기 때문에 클래스 수를 늘림으로써 복잡성이 증가한다.
- 주입된 객체들에 관한 코드 추적이 어렵다.
- 초기 개발 노력이 필요하다.
- 의존성 주입 프레임워크를 사용하면 빌드 시간이 늘어날 수 있으며, 프레임워크에 대한 의존도를 높인다.

### 의존성 주입의 3가지 방법

- **필드 주입:** 멤버 변수에 `@Autowired`를 바로 붙이는 방식
- **수정자 주입:** Setter 메서드를 통해 주입 (객체가 중간에 변경될 위험이 있음)
- **생성자 주입:** 생성자를 통해 주입받는 방식
    - **AppConfig**
        - 애플리케이션의 전체 동작 방식을 구성하기위해 구현 객체를 생성하고, 연결하는 책임을 가지는 별도의 설정 클래스 생성
            - AppConfig에 설정을 구성한다는 뜻의 @Configuration을 붙임
                
                ```sql
                @Configuration
                public class AppConfig{
                ```
                
            - 메서드에 @Bean을 붙여준다. 이렇게 하면 스프링 컨테이너에 스프링 빈으로 등록
            - AppConfig에 실제 동작에 필요한 구현 객체를 생성
            - AppConfig는 생성한 객체 인스턴스의 참조(레퍼런스)를 생성자를 통하여 주입(연결)한다.
        - 객체의 생성과 연결은 AppConfig가 담당
            - 컴포넌트 스캔 (Component Scan)
                - AppConfig에 객체를 하나하나 등록하기 귀찮고 복잡해지기 때문에, 스프링은 @ComponentScan이라는 기능을 제공
                - 클래스에 @Component (또는 @Service, @Controller 등)를 붙여두면 스프링이 알아서 클래스들을 탐색해 DI 컨테이너에 등록하고 주입
